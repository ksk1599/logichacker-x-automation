# Knowledge Agent — 도메인 교정 규칙

> 이 파일에 있는 내용은 도메인 지식 오류 교정 목록이다.
> Writer나 QA가 잘못된 주장을 하면 이 파일 기준으로 즉시 반박하라.

---

## [교정 #1] 키워드 위치(앞/뒤)는 가점 없음

### 잘못된 주장 (목격된 사례)
- "상품명 앞에 온 키워드에 2배 이상 가점을 준다"
- "앞에 올수록 가점 3배, 뒤에 올수록 0.5배"
- "중요 키워드는 맨 앞에 써야 유리하다"

### 올바른 사실
- 키워드 위치(앞/뒤)는 검색 순위에 **영향 없음**
- 중요한 것: 키워드끼리의 **배열 관계** (얼마나 가까이, 순서대로 붙어있는지)
- 근거: `skills/knowledge/free.md` — "위치(앞/뒤)의 진실 — 흔한 낭설 부수기"

### 올바른 표현
- "키워드 배열 관계(가까이 붙이기)가 중요하다"
- "위치가 아니라 배열이 핵심이다"
- "키워드끼리 손을 맞잡고 있어야 한다"

## Knowledge Agent 행동 지침

1. Writer/QA로부터 질의가 오면 이 파일을 먼저 확인한다.
2. 질의 내용이 위 교정 목록과 충돌하면 **즉시 오류로 표시**하고 올바른 내용 반환.
3. 오류 발견 시 응답 형식:
```json
{
  "answer": "틀렸습니다. 키워드 위치는 가점에 영향 없습니다.",
  "correction": "domain_corrections.md #교정1",
  "correct_expression": "배열 관계(가까이 붙이기)가 중요하다"
}
```
