# QA Agent — 도메인 정확성 검수 규칙

> 기본 4줄 형식 체크 외에 도메인 정확성도 반드시 검수한다.
> 이 파일에 있는 금지 주장이 글에 있으면 **즉시 반려(fail)**한다.

---

## 도메인 오류 체크리스트

아래 표현이 글에 한 글자라도 포함되면 형식이 완벽해도 반려한다.

| 금지 표현 | 이유 | 올바른 방향 |
|---|---|---|
| "앞에 올수록 가점" | 사실 아님 | 위치 무관, 배열 관계가 중요 |
| "앞에 온 키워드에 N배" | 사실 아님 | 동일 |
| "중요 키워드는 맨 앞에" | 낭설 | 동일 |
| "배열 순서가 검색량 결정" | 오해 유발 | 배열 관계(가까이 붙기) |
| "가중치" (랭킹 맥락) | 용어 오류 | "가점"으로 교체 |

---

## 도메인 의심 시 처리

1. 글에 스마트스토어 알고리즘 관련 수치/주장이 있으면 → Knowledge Agent에 확인
2. Knowledge Agent가 오류 확인 시 → 즉시 `fail` 처리
3. 피드백에 반드시 올바른 표현 제시

---

## 반려 피드백 예시 (도메인 오류)

```json
{
  "verdict": "fail",
  "issues": [
    {
      "line": 3,
      "rule": "도메인 정확성",
      "problem": "'앞에 올수록 가점 3배'는 사실이 아닙니다",
      "fix": "'키워드끼리 가까이 붙어야 가점'으로 수정",
      "source": ".claude/rules/qa/check_rules.md"
    }
  ]
}
```
