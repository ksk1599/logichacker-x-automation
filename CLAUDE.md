# 로직해커 엑스 자동화 프로젝트 — AI 지침서

## 도메인 지식 교정 (절대 규칙)

### ❌ 키워드 위치(앞/뒤) 가점 — 낭설, 절대 쓰지 마라

> **"상품명에서 키워드가 앞에 올수록 가점이 높다"는 말은 완전히 틀렸다.**

- Writer, QA, 모든 에이전트가 이 낭설을 콘텐츠에 쓰면 **즉시 반려**한다.
- 실제 테스트 결과: 키워드가 맨 앞이든 중간이든 뒤든 검색 순위에 차이 없음.
- 중요한 것은 **위치(앞/뒤)가 아니라 배열 관계** (키워드끼리 얼마나 가까이, 순서대로 붙어있는지).

**잘못된 표현 예시 (절대 사용 금지):**
- "앞에 올수록 가점 3배"
- "앞에 온 키워드에 2배 이상 가점을 준다"
- "중요 키워드는 맨 앞에 써야 유리하다"

**올바른 표현:**
- "키워드 배열 관계(얼마나 가까이 붙어있는지)가 가점에 영향을 준다"
- "위치가 아니라 배열이 핵심이다"

## 용어 규칙

### "가중치" → "가점"으로 표현할 것

- 스마트스토어 상위노출 맥락에서 "가중치"라는 말 대신 **"가점"**을 사용한다.
- 단, ML/알고리즘 기술적 맥락(예: 과거 Linear 모델 설명)에서는 예외 가능.
