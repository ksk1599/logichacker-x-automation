Why vanilla?

왜 바닐라 JS인가

  • 외부 의존성 제로 — 번들 크기 0KB
  • 브라우저 표준만으로 완전한 구현
  • 커스터마이즈의 자유도 극대화
  • 단일 HTML 파일로 배포 가능
Features

핵심 기능

  • 풀스크린 뷰포트 & 반응형 레이아웃
  • 키보드 / 터치 / 스와이프 네비게이션
  • GPU 가속 슬라이드 전환 애니메이션
  • 순차 등장 reveal 효과
  • 코드 블록 라인 하이라이트
Architecture

아키텍처

레이아웃
fixed 뷰포트 + absolute 슬라이드

전환
CSS Transition + transform/opacity

상태 관리
slideIndex 단일 상태 + URL hash

등장 효과
.reveal 클래스 + transition-delay

Layout

CSS 레이아웃 전략

// 풀스크린 뷰포트

position: fixed 뷰포트 안에
position: absolute 슬라이드를 겹쳐
transform: translateX로 전환합니다

// 콘텐츠 영역

max-width: 680px로 읽기 편한 폭 유지
clamp()로 반응형 타이포그래피 처리

HTML 구조

<div class="slide-viewport">
  <section class="slide active">
    <div class="slide__inner">
      <h2 class="reveal">제목</h2>
      <p class="reveal">내용</p>
    </div>
  </section>
</div>

.reveal 클래스로 순차 등장 효과가 자동 적용됩니다

슬라이드 전환 로직

goNext() {
  if (this.isAnimating) return;
  this._goSlide(this.current + 1, 'next');
}

// GPU 가속: transform + opacity만 사용
to.style.transform = 'translateX(0)';
from.style.transform = 'translateX(-100%)';

isAnimating 플래그 + transitionend로 이벤트 잠금

Pitfalls

주의사항

  • display:none 사용 금지 — transition 불가
  • 모바일 높이는 100dvh 사용
  • will-change는 동적 토글 (정적 적용 시 메모리 낭비)
  • transitionend 미발생 대비 setTimeout fallback 필수
  • 터치 passive:false에서만 preventDefault 가능
