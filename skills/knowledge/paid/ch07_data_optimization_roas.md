# 7강 — 데이터 최적화: 제외 키워드 관리와 ROAS 극대화 (심화)

## 핵심 메시지
- 광고를 '돌리는' 수준에서 '지배하는' 수준으로 올라가라
- 1그룹 1키워드 전략으로 돈이 되는 키워드와 돈만 먹는 키워드를 철저히 분리하라
- 광고에서 배열 최적화는 의미 없다 — 오직 입찰가 싸움이다

---

## 중요: 광고에서의 검색인식영역

### 광고 검색인식영역 7가지
1. 기본 상품명 (혹은 노출용 상품명)
2. 스토어명
3. 브랜드
4. 제조사
5. 카테고리
6. 속성
7. 태그

### ⚠️ 광고는 자연노출과 다르다

> **광고에서는 가점이 없다. 오직 입찰가 싸움이다.**

- 검색인식영역에 키워드가 있기만 하면, 그 뒤는 순전히 입찰가 경쟁
- 노출용 상품명에 키워드가 앞/뒤로 배치되어도 가점 차이 없음
- 중복어를 쓸 이유도 없음 — 자리수만 차지할 뿐

**예시:**
- A 노출용 상품명: "빨간 노란 초록 안경"
- B 노출용 상품명: "빨간 초록 안경" + 태그: #초록
- '초록 안경' 검색 시 → A와 B의 가점 동일 (자연노출은 A가 유리할 수 있지만 광고에서는 동일)

## 1:1 매칭 설계 — 문제와 해결

### 문제: 통제 불능의 늪

네이버 광고는 **키워드별 입찰가 설정이 불가능**하다.

| 키워드 | 상황 | 설정 입찰가 |
|---|---|---|
| 빨간 안경 (메인, 전환율 高) | 1,000원 써야 함 | 500원 (기회 손실) |
| 노란 안경 (경쟁 심함) | 500원 | 500원 |
| 빨간 선글라스 안경 (관련 없음) | 전환율 0% | 500원 (돈 낭비) |

→ 전교 1등(효자 키워드)과 꼴등(불필요 키워드)에 똑같은 학원비(500원)를 주는 꼴

### 해결: 1:1 매칭 설계 실행법

```
📁 캠페인
  📂 그룹: "안경" (키워드명 동일)
    📄 소재: "안경" 키워드 최적화 1개
    → 제외 키워드: "안경" 제외 나머지 모두 차단 (140개)
  📂 그룹: "뿔테안경"
    📄 소재: "뿔테안경" 키워드 최적화 1개
    → 제외 키워드: "뿔테안경" 제외 나머지 모두 차단
  ...
```

**실제 키워드 설계 예시 (아이템스카우트 활용):**

| 순위 | 키워드 | 검색량 | 그룹명 |
|---|---|---|---|
| 1 | 안경 | 66,090 | ㄴ_안경 |
| 2 | 안경테 | 38,690 | ㄴ_안경테 |
| 3 | 뿔테안경 | 22,450 | ㄴ_뿔테안경 |
| 4 | 무테안경 | 10,980 | ㄴ_무테안경 |
| 5 | 명품안경테 | 8,610 | ㄱ_명품안경테 |
| ... | ... | ... | ... |

> 그룹명 정렬 팁: 1,000단위 검색량 → 앞에 'ㄱ', 10,000단위 → 'ㄴ' 표기 (로파고 정렬법)

**소재 개수:**
- 15개 그룹 생성 (타겟 키워드가 20개면 20개)
- 10개 ON / 나머지 5개는 예비용 OFF
- 필요 시 즉시 ON 전환 가능하도록 예비 준비

## 140개 제외 키워드 등록법

### 중복 없이 등록하기

| 방법 | 그룹 제외 | 소재 제외 | 실제 차단 수 |
|---|---|---|---|
| 나쁜 예 (중복) | a, b, c | c, d, e | 5개 (c 중복 낭비) |
| 좋은 예 (분리) | a, b, c | d, e, f | 6개 (완전 차단) |

### 실전 등록 순서

1. 아이템스카우트에서 키워드 엑셀 추출
2. 검색량 내림차순 정렬
3. 상위 71개 복사 → **광고 소재**의 '제외 검색어 추가'에 붙여넣기
4. 타겟 키워드(예: '안경')는 제외 목록에서 삭제 후 저장 (71개 복사해서 1개 빼면 70개)
5. 그 다음 71개 복사 → **광고 그룹**의 제외 검색어에 붙여넣기
6. 검색 네트워크 영역 → '사용'으로 변경 후 저장

## ROAS 계산 및 관리

### 손익분기점 ROAS 공식

```
손익분기점 ROAS = (마진 / 판매가) × 100
```

**예시:**
- 판매가: 20,000원
- 원가: 3,000원 → 마진: 17,000원
- 손익분기점 ROAS = (17,000 / 20,000) × 100 = **117.6%**
- → 매번 ROAS가 117.6% 이상인지 확인

### 입찰가 산정 방법

| 방법 | 설명 | 추천 |
|---|---|---|
| 낮은 입찰가 → 점차 올리기 📈 | 보수적, 손실 적음 | 보수적 성향 |
| 높은 입찰가 → 점차 내리기 📉 | 처음 손실 있지만 빠르게 상위권 파악 | **권장** |

- ROAS를 잘 기록하고 유동적으로 입찰가 수정
- 아무리 좋은 키워드라도 지속 적자 ROAS면 과감하게 삭제하는 강단 필요

## 도메인 주의사항 (광고 vs 자연노출)
- 자연노출: 키워드 배열 관계(가까이 붙이기)가 가점에 영향
- 광고: 배열 관계 가점 없음. 오직 입찰가 경쟁
- Writer/QA는 이 두 맥락을 절대 혼동하지 마라
